(一)

**图灵完备**是指一个计算系统（比如一种编程语言、一个指令集或一台机器）**具有模拟通用图灵机的所有计算能力**。（通用图灵机就是一个可以读写纸带和条件分支跳转的机器）
通俗地讲：**只要是现实中可以通过计算解决的问题，图灵完备的系统都能解决。** （前提是给它无限的内存和无限的时间）。
一个系统要达到图灵完备，通常只需要具备两个极其基础的核心能力：

1. **数据操作（状态改变）**：能够读取、修改和存储数据（比如有内存、有寄存器，能做加减法或简单的位运算）。
2. **条件控制流（分支跳转）**：能够根据当前的数据状态做出不同的决定，并跳转到不同的步骤执行（类似于if...else，或者 Jump if Greater这种条件跳转指令），从而实现**循环**。

如何证明我们的vspm是否是图灵完备的？
如果它**同时具备**以下三种指令，那它在实际意义上就是图灵完备的：

1. **内存/寄存器操作指令**：能把数据从内存读到寄存器，能把寄存器的数据写回内存。
2. **基本运算指令**：至少包含一种运算。比如加法（ADD）、减法（SUB），或者甚至只需要一个按位与非（NAND）指令。
3. **条件跳转指令（关键！）**：必须有一种指令能根据条件打乱顺序执行。

那么众所周知，我们的原型机vspm确实拥有以上三种指令（mov类，add和sub，jmp和jg），所以，我们的vspm确实是图灵完备的！

（三）

什么是CPU的中断机制？
**让 CPU 在执行当前程序时，能够被“打断”，去优先处理更紧急的事件** 。

为什么现代CPU需要中断机制？
回答这个问题之前，我们先来看没有中断机制的vspm是怎么做的。拿a-inst举例，在我们按下si之后，vspm会执行IN Rd指令，也就是等待我们外部输入数据，那么这时候cpu会不断地去问我们的键盘：你输入数据了吗？对于一个好的cpu，哪怕我们迅速就按下键盘，在这零点几秒里它也会白白循环几百万次，很费电。
所以现代CPU引入了中断机制，目的无非有三：

1. **让CPU不白等（告别“轮询死等”）**
2. **实现实时响应与异常处理：**
   /*如果程序运行中发生了紧急的事情（比如突然断电），如果没有中断机制，CPU 就会死机。中断机制可以立刻强行打断 CPU，跳转到应急程序进行抢救或报错。*/
3. **实现多任务操作系统（时间片轮转）：**
   /*现在我们能在电脑上一边打游戏一边听音乐，全靠主板上的“定时器硬件”。它每隔几毫秒就会给 CPU 发送一次**时钟中断**，强制 CPU 挂起当前程序，切换到另一个程序。可以说，**没有中断，就没有现代操作系统。***/

如果要实现中断，vspm需要增加哪些硬件？

1. **中断请求引脚**：
   在 CPU 芯片外部增加一个物理引脚，接收外部设备（如键盘、定时器）发来的高低电平信号。
2. **中断允许触发器**：
   在控制器内部增加一个 1 bit 的标志位硬件。如果 IE=0，CPU 会屏蔽外界的打扰（比如在执行极其关键的原子操作时）；IE=1 时才允许中断。
3. **硬件堆栈指针寄存器**
   当 CPU 被打断时，它必须把当前正在执行的地址（程序计数器 PC 的值）以及状态位（比如 SUB 指令产生的 G 标志位）找个地方“藏起来”，也就是**保护现场**。这需要一块专用的内存区域（堆栈）和一个专门指向堆栈顶部的寄存器 SP。目前的 vspm 没有堆栈概念，这是必须加上的。
4. **中断向量生成逻辑：
   CPU 被打断后，下一条指令该去哪里执行？硬件需要能自动生成一个固定的内存地址（比如跳转到 RAM 的 0x0008 地址），这个地址存放着专门处理中断的代码（中断服务程序 ISR）。
